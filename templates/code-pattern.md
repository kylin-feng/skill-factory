## 审查流程

用户提供代码（粘贴/文件路径/PR链接）后：

### Step 1: 快速扫描
扫描所有🔴规则，列出违规项：
```
🔴 发现 N 个必须修复的问题：
1. [行号/位置] [规则名]：[说明]
2. ...
```

### Step 2: 详细分析
对每个违规项：
- 指出具体位置（文件:行号）
- 解释为什么违规
- 给出修改后的代码

### Step 3: 改进建议
扫描🟡和🟢规则，提供建议（不强制）

### Step 4: 总结报告
```
审查完成：
🔴 必须修复：N 项
🟡 建议修复：N 项
🟢 最佳实践提示：N 条

[通过/需要修改后重新审查]
```
